Если вы уже исправили ошибку с localtime и настроили проект, то процесс запуска выглядит так

1. Как запустить прямо сейчас (горячие клавиши)

F5 — Запуск в режиме отладки (медленнее, но можно искать ошибки).

Ctrl + F5 — Запуск без отладки (быстрее, программа просто откроется как обычное приложение).

2. Важный нюанс два задания — две функции main

У вас два кода (График и Часы). В одном проекте в C++ может быть только одна функция main. Если вы вставили оба кода в один файл или создали два файла в одном проекте, программа не скомпилируется (выдаст ошибку main уже определен).

Как переключаться между ними

Простой способ Откройте ваш файл main.cpp, удалите старый код, вставьте новый (для другого задания) и нажмите F5.

Способ для профи (если файлов два)

Нажмите правой кнопкой на тот файл, который не нужен сейчас, в обозревателе решений.

Выберите Свойства.

В пункте «Исключить из сборки» (Excluded from Build) выберите Да.

Теперь компилятор будет видеть main только в одном файле.

3. Что делать, если окно открывается и сразу закрывается

Если вы запускаете через F5 и программа закрывается мгновенно

Убедитесь, что внутри функции while (!glfwWindowShouldClose(window)) есть код.

Если это происходит из-за ошибки (например, glfwInit не сработал), попробуйте нажать Ctrl + F5. Окно консоли останется открытым, и вы сможете прочитать текст ошибки.

4. Где найти готовую программу (.exe)

Если вы хотите отправить этот файл преподавателю или другу

Посмотрите в папку вашего проекта.

Там будет папка x64 (или Debug  Release).

Внутри лежит файл имя_проекта.exe.
Примечание Чтобы он работал на другом компьютере, рядом с ним должна лежать библиотека glfw3.dll (если вы использовали динамическое подключение).

Краткий чек-лист перед нажатием F5

В самом верху #define _CRT_SECURE_NO_WARNINGS.

В Свойствах проекта - Компоновщик - Ввод добавлено opengl32.lib.

NuGet пакет glfw установлен.

Тип сборки (сверху в центре) стоит x64.
